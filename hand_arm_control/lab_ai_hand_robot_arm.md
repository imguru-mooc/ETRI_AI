# 실습 — AI 손 제어 로봇팔 (회전·상하·집게) 🦾

> **멀티모달 인지 기반 피지컬 AI 로봇 시스템 구현 과정**
> 카메라로 손을 인식(MediaPipe Hands)해서, **손을 움직이는 대로 로봇팔 전체**를 제어합니다.
> 앞선 "손가락 집게 제어"를 확장해, **회전(베이스)·상하(어깨)·집게**를 한 손으로 동시에 조종합니다.

---

## 1. 무엇을 만드나요?

```
   손(팔) 동작                AI 인식            BLE 전송            로봇팔
 ┌───────────┐   카메라  ┌──────────┐  Web BT  ┌──────────┐      ┌──────────┐
 │ 좌우 이동  │ ───────▶ │ MediaPipe│ ───────▶ │  포니봇   │ ───▶ │ S1 회전   │
 │ 상하 이동  │          │  Hands   │  Nordic  │ ble_lib  │      │ S2 상하   │
 │ 엄지·검지  │          │ 21 점 추적│   UART   │ on_write │      │ S4 집게   │
 └───────────┘          └──────────┘          └──────────┘      └──────────┘
```

### 손동작 ↔ 로봇팔 관절 매핑

| 손동작 | 측정 방법 | 제어 관절 | 각도 범위 |
|--------|-----------|-----------|-----------|
| 손을 **좌우로** 이동 | 손바닥(9번 점) X 위치 | **S1 베이스 = 회전** | 10~170° |
| 손을 **위아래로** 이동 | 손바닥(9번 점) Y 위치 | **S2 어깨 = 상하** | 30~150° |
| **엄지·검지** 벌리고 오므림 | 4번·8번 점 거리 | **S4 집게** | 0~90° |
| (팔꿈치 S3) | 웹앱 슬라이더 | 보조 보정 | 0~180° |

> 핵심 아이디어: **손 = 로봇팔의 분신.** 손을 움직이는 위치가 곧 팔의 자세가 됩니다.

## 2. 준비물 & 파일

- 조립된 **로봇팔** (서보 S1·S2·S3·S4 연결, 교재 186p~)
- 포니봇 보드 (BLE 가능), Thonny
- 보드에 올라가 있어야 할 패키지: `ble_library`, `ai_ponybot`
- 제공 파일 2개
  - `ai_arm_ble.py` — **포니봇(보드)** 에서 실행하는 BLE 수신 펌웨어
  - `hand_arm_control.html` — **PC/폰** 브라우저에서 여는 AI 제어 웹앱

> 🔎 사전 점검 (REPL):
> ```python
> import ble_library, bluetooth          # 에러 없으면 OK
> from ai_ponybot import i2c, PonyServo  # OK
> ```

## 3. 따라하기

### STEP 1. 로봇팔 영점 맞추기 (조립 직후 1회)

서보가 90°일 때를 기준으로 조립되어야 동작이 자연스럽습니다. 교재 194p 코드로 모든 관절을 90°에 둔 뒤 팔을 곧게 정렬하세요.

```python
from ai_ponybot import i2c, PonyMotor, PonyServo
motor = PonyMotor(i2c); servo = PonyServo(motor.pwm)
for ch in (1, 2, 3, 4):
    servo.set_angle(ch, 90)
```

> 집게(S4)는 0~90 범위이므로 90이면 "활짝 열림"입니다.

### STEP 2. 보드에 펌웨어 실행

1. Thonny에서 `ai_arm_ble.py`를 열고 **보드에 저장** 후 실행(▶).
2. OLED에 `BLE WAITING / name: ESP-ARM`, 본체 LED가 **빨강**이면 정상 대기 상태입니다.

### STEP 3. 웹앱 열고 블루투스 연결

1. `hand_arm_control.html`을 **HTTPS 주소**로 엽니다. (GitHub Pages 권장 — 아래 배포 팁)
2. **🔵 블루투스 연결** → 목록에서 **`ESP-ARM`** 선택 → "연결됨" 표시 확인.
   - 이때 보드 OLED는 `BLE CONNECTED`, LED는 **초록**으로 바뀝니다.

### STEP 4. 카메라 켜고 손 인식 확인

1. **📷 카메라 시작** → 손을 화면에 비춥니다.
2. 손 위에 초록 선·노란 점(21개 랜드마크)이 그려지고 "손 인식: 인식 중"이 뜨면 성공.
3. 아직 로봇팔이 크게 움직일 수 있으니 **손을 화면 한가운데**에 두고 시작하세요.

### STEP 5. 회전(베이스) 익히기 🔄

손을 **좌 ↔ 우**로 천천히 움직여 보세요. 베이스(S1)가 따라 회전합니다.
오른쪽 패널 **🔄 회전** 막대와 각도 숫자가 함께 변합니다.

> 방향이 반대로 느껴지면 **"회전 반전"** 체크박스를 켜세요. (코드 수정 불필요)

### STEP 6. 상하(어깨) 익히기 ↕️

손을 **위 ↔ 아래**로 움직여 어깨(S2)를 올리고 내려 보세요.
보통 손을 위로 들면 팔도 위로 올라가도록 설정돼 있습니다. 반대면 **"상하 반전"** 체크.

### STEP 7. 집게 + 통합 동작 (물건 집기) 🤏

1. 엄지·검지를 **오므리면** 집게가 닫히고, **벌리면** 열립니다.
2. 이제 셋을 합쳐 봅니다:
   - 손을 옮겨 팔을 **물건 위로** (회전+상하)
   - 팔꿈치 슬라이더로 높이 미세 조정
   - 손가락을 **오므려** 집기 → 손을 옮겨 **다른 곳으로** → **벌려** 놓기
3. 자세가 흐트러지면 **🏠 HOME** 버튼으로 기본 자세(모두 90°, 집게 45°)로 복귀.

## 4. 코드 핵심 해설

### (1) 손 위치 → 각도로 바꾸기 (웹앱)

손바닥 기준점은 **9번(중지 MCP)** 을 씁니다. 핀치(4·8번)와 **겹치지 않아** 위치 제어가 안정적입니다.

```javascript
const px = lm[9].x, py = lm[9].y;          // 손 위치(0~1)
const B = mapClamp(px, 0.15, 0.85, 10, 170, invBase);  // 좌우 -> 회전
const S = mapClamp(py, 0.15, 0.85, 30, 150, !invSh);   // 상하 -> 어깨
```

화면 가장자리(0~0.15, 0.85~1)는 잘라내 흔들림을 줄이고, **EMA 평활**로 떨림을 부드럽게 합니다.

```javascript
cur.B += (target.B - cur.B) * 0.30;        // 0.30이 작을수록 더 부드럽고 느림
```

### (2) 4관절을 한 번에 전송

```javascript
// 예: "B:120,S:80,E:90,G:30"  (변화 있을 때 & 80ms 간격으로만 전송)
const s = `B:${B},S:${S},E:${E},G:${G}`;
```

### (3) 보드에서 받아 서보로 (펌웨어)

교재의 BLE 수신 패턴(`on_write`) 그대로, 문자열을 쪼개 각 서보에 적용합니다.

```python
def on_rx(v):
    data = v.strip()                       # "B:120,S:80,E:90,G:30"
    for part in data.split(','):
        j, val = part.split(':')           # 'B','120'
        if j in CH:
            apply(j, float(val))           # servo.set_angle(채널, 각도)
```

`apply()` 안에서 관절별 **안전 범위로 clamp** 후 `servo.set_angle()`을 호출합니다.

## 5. 도전과제 🎯

> ### 🎯 도전 1 — 두 손으로 4축 제어 (팔꿈치까지)
> `maxNumHands:2`로 바꾸고, **왼손 = 회전·상하·집게**, **오른손 높이 = 팔꿈치(S3)** 로 매핑해
> 슬라이더 없이 팔 전체를 손만으로 움직여 보세요.
> 힌트: `results.multiHandedness`로 좌/우 손을 구분합니다.

> ### 🎯 도전 2 — '집기 모드' 잠금
> 주먹을 1초간 쥐면(핀치 최소 유지) 현재 회전·상하를 **고정(lock)** 하고 집게만 동작하게 만들어,
> 흔들림 없이 정확히 집도록 개선해 보세요. (OLED에 `LOCK` 표시)

> ### 🎯 도전 3 — 시퀀스 녹화 & 재생
> HOME→집기→이동→놓기 동작을 배열에 **녹화**해 두고, 버튼 한 번으로 **자동 재생**하는
> "한 번 가르치면 반복하는 로봇팔(Teach & Repeat)"을 만들어 보세요.

> ### 🎯 도전 4 — 포즈 인식 결합
> MediaPipe **Pose**(상체)로 바꿔, 실제 팔을 들어 올리면 로봇팔이 따라 하는
> "내 팔의 그림자 로봇"으로 확장해 보세요.

> ### 🎯 도전 5 — 안전 속도 제한
> 한 번에 각도가 너무 크게 변하면 위험합니다. 프레임당 변화량을 `MAX_STEP`(예: 3°)로 제한하는
> **속도 제한(slew rate)** 을 보드 또는 웹앱에 추가해 보세요.

## 6. 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| 연결 목록에 `ESP-ARM` 없음 | 보드 미실행/이름 불일치 | `ai_arm_ble.py` 실행 확인, 코드의 장치명 일치 |
| 버튼이 회색(연결 불가) | HTTP로 열림 | **HTTPS**(GitHub Pages)로 열기 |
| 손은 인식되는데 팔이 안 움직임 | BLE 미연결 | "블루투스 연결" 먼저 → "연결됨" 확인 |
| 방향이 반대 | 서보 장착 방향 | 해당 **반전 체크박스** 사용 |
| 팔이 덜덜 떨림 | 평활 부족/전원 | `SMOOTH` 값 낮추기, 서보 전원 4.8V↑ 확인 |
| 팔이 갑자기 크게 점프 | 손이 화면 끝에서 등장 | 손을 **가운데**에서 시작, 도전 5(속도제한) 적용 |
| iPhone에서 안 됨 | iOS 미지원 | Chrome/Edge(데스크톱·안드로이드) 사용 |

## 안전 수칙 ⚠️

- 로봇팔 **가동 범위 안에 손·물건**을 두지 마세요. (끼임 주의)
- 처음엔 **손을 천천히** — 익숙해지면 속도를 올립니다.
- 서보가 떨거나 멈추면 전원(최소 4.8V) 부족일 수 있습니다. (교재 192p)
- 비상 시 **HOME** 또는 Thonny **Ctrl+C** 로 정지.

> 이 실습으로 **AI 인식 → 무선 통신(BLE) → 다축 액추에이터 제어**라는
> 피지컬 AI의 전체 파이프라인을 손으로 직접 체험하게 됩니다.
