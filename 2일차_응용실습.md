## 0. 공통 준비 (모든 실습 공통)

### 0-1. 하드웨어 핀·주소 요약

| 구분 | 부품 | 연결 | 비고 |
|------|------|------|------|
| 출력 | NeoPixel 5×5 (25구) | `Pin(13)` | `np[i]=(R,G,B)` 후 **`np.write()`** 필수 |
| 출력 | 메카넘 DC 모터 ×4 | I2C (PWM 드라이버) | `PonyMotor` |
| 출력 | 서보 8채널 | I2C (PWM 공유) | `PonyServo` |
| 출력 | OLED | I2C | `PonyOLED` |
| 입력 | 버튼 A / B | `Pin(36)` / `Pin(39)` | 누르면 `value()==0` |
| 입력 | 마이크(소리) | `ADC(Pin(35))` | `max-min`으로 크기 측정 |
| 입력 | 가속도(MPU6050) | I2C `0x69` | `imu.get_values()` |
| 입력 | 온도 | 칩 내장 | `esp32.mcu_temperature()` (℃) |

### 0-3. I2C 버스 공유 — 가장 중요한 개념

1일차에는 가속도 센서용 I2C를 직접 만들었지만(`I2C(0, scl=Pin(21), sda=Pin(22))`),
이번 통합 실습에서는 **모터·OLED·가속도가 같은 I2C 버스**를 씁니다.
그래서 `ai_ponybot`이 만들어 준 `i2c` 하나를 **모두에게 공유**합니다.

```python
from ai_ponybot import i2c, PonyMotor, PonyOLED   # i2c를 가져와서
import mpu6050
imu   = mpu6050.accel(i2c, 0x69)   # 가속도도 같은 i2c 사용
motor = PonyMotor(i2c)
oled  = PonyOLED(i2c)
```

### 0-4. 안전 수칙 ⚠️

- **LED 밝기는 낮게**: `(10, 0, 0)`처럼 0~20 사이. 큰 값은 전류↑·눈부심.
- **주행 속도는 낮게 시작**: 40~60. 책상 위 실습은 **바퀴를 들거나** 바닥에서.
- **떨어짐 주의**: 책상 끝에서 멀리. 비상 정지는 `motor.mecanum(5)` 또는 `motor.drive('stop')`.
- 코드 정지: Thonny에서 **Ctrl+C** (또는 Stop 버튼).

## 실습 1. 기울기 조종 메카넘카 🕹️

> **보드를 기울이는 방향으로 로봇이 달린다.** 가속도 센서를 운전대로!

**감지** 가속도(기울기) → **판단** 방향·속도 매핑 → **동작** 메카넘 주행 + LED 화살표 + OLED 표시

### STEP 1. 가속도 값부터 눈으로 확인하기

먼저 기울일 때 어떤 값이 나오는지 셸에서 관찰합니다.

```python
from ai_ponybot import i2c
import mpu6050
from time import sleep

imu = mpu6050.accel(i2c, 0x69)
while True:
    v = imu.get_values()
    print('X=%6d  Y=%6d' % (v['AcX'], v['AcY']))
    sleep(0.2)
```

보드를 앞/뒤로 기울이면 `AcY`, 좌/우로 기울이면 `AcX`가 ±로 크게 변합니다.
평평할 때 값이 0 근처가 되도록 **기준값**을 머릿속에 기억해 두세요. (보통 ±2000 이하는 "평평")

### STEP 2. 기울기 → 방향으로 바꾸는 함수 만들기

```python
DEAD = 4000   # 이 값보다 작은 기울기는 '정지'로 무시 (감도)

def tilt_to_dir(x, y):
    if abs(x) < DEAD and abs(y) < DEAD:
        return 'stop'
    if abs(y) >= abs(x):
        return 'fwd' if y > 0 else 'back'
    else:
        return 'right' if x > 0 else 'left'
```

> 💡 부호(`y>0`)가 실제 기울임과 반대면 `>`를 `<`로 바꾸세요. 보드 장착 방향에 따라 달라집니다.

### STEP 3. 방향별 LED 화살표 패턴 준비

```python
# tilt_car.py  ----  실습 1 전체 코드
from ai_ponybot import i2c, PonyMotor, PonyOLED
from neopixel import NeoPixel
from machine import Pin
import mpu6050
from time import sleep

imu   = mpu6050.accel(i2c, 0x69)
motor = PonyMotor(i2c)
oled  = PonyOLED(i2c)
np    = NeoPixel(Pin(13, Pin.OUT), 25)

DEAD  = 4000
CODE  = {'fwd': 8, 'back': 2, 'left': 4, 'right': 6, 'stop': 5}  # mecanum 코드
LABEL = {'fwd': 'FORWARD', 'back': 'BACK', 'left': 'LEFT',
         'right': 'RIGHT', 'stop': 'STOP'}
ARROW = {
    'fwd':   [2, 6,7,8, 10,12,14, 17, 22],
    'back':  [2, 7, 10,12,14, 16,17,18, 22],
    'left':  [2, 6, 10,11,12,13,14, 16, 22],
    'right': [2, 8, 10,11,12,13,14, 18, 22],
    'stop':  [12],
}

def tilt_to_dir(x, y):
    if abs(x) < DEAD and abs(y) < DEAD:
        return 'stop'
    if abs(y) >= abs(x):
        return 'fwd' if y > 0 else 'back'
    return 'right' if x > 0 else 'left'

def show_arrow(name):
    for i in range(25):
        np[i] = (0, 0, 0)
    for i in ARROW[name]:
        np[i] = (0, 10, 0)
    np.write()

print('기울기 조종 시작! (Ctrl+C로 정지)')
last = None
while True:
    v = imu.get_values()
    d = tilt_to_dir(v['AcX'], v['AcY'])
    motor.mecanum(CODE[d], 60)

    if d != last:            # 방향이 바뀔 때만 화면/LED 갱신
        show_arrow(d)
        oled.clear()
        oled.write_line(0, 'TILT DRIVE')
        oled.write_line(1, LABEL[d])
        oled.show()
        last = d
    sleep(0.05)
```

## 실습 2. 박수 리듬 댄스봇 👏

> **박수 횟수로 안무를 지휘한다.** 마이크가 리모컨!

**감지** 마이크(박수 수) → **판단** 횟수 → 안무 선택 → **동작** 메카넘 댄스 + LED 비트 + OLED

### STEP 1. 박수(소리) 한 번을 감지하기

소리 크기 = `max - min`. 임계값을 넘으면 "박수 1회"로 셉니다.

```python
from machine import ADC, Pin
from time import sleep, ticks_ms, ticks_diff

mic = ADC(Pin(35))
mic.atten(ADC.ATTN_11DB)

def sound_level():
    s = [mic.read() for _ in range(50)]
    return max(s) - min(s)

THRESH = 1500   # 박수 인식 기준 (환경에 맞게 조절)
while True:
    lv = sound_level()
    if lv > THRESH:
        print('박수!', lv)
        sleep(0.3)      # 같은 박수를 두 번 세지 않도록 잠깐 대기
    sleep(0.02)
```

### STEP 2. 일정 시간 동안 박수 횟수 세기

"2초 동안 몇 번 쳤나"를 세서 안무를 고릅니다.

```python
def count_claps(window_ms=2000):
    count = 0
    t0 = ticks_ms()
    while ticks_diff(ticks_ms(), t0) < window_ms:
        if sound_level() > THRESH:
            count += 1
            sleep(0.3)      # 디바운스
        sleep(0.02)
    return count
```

### STEP 5. 전체 합치기

```python
# clap_dance.py  ----  실습 2 전체 코드
from ai_ponybot import i2c, PonyMotor, PonyOLED
from neopixel import NeoPixel
from machine import ADC, Pin
from time import sleep, ticks_ms, ticks_diff
import random

motor = PonyMotor(i2c)
oled  = PonyOLED(i2c)
np    = NeoPixel(Pin(13, Pin.OUT), 25)
mic   = ADC(Pin(35)); mic.atten(ADC.ATTN_11DB)

THRESH = 1500

def sound_level():
    s = [mic.read() for _ in range(50)]
    return max(s) - min(s)

def count_claps(window_ms=2000):
    count, t0 = 0, ticks_ms()
    while ticks_diff(ticks_ms(), t0) < window_ms:
        if sound_level() > THRESH:
            count += 1
            sleep(0.3)
        sleep(0.02)
    return count

def beat():
    c = random.choice([(12,0,0),(0,12,0),(0,0,12),(10,10,0)])
    for i in range(25): np[i] = c
    np.write(); sleep(0.15)
    for i in range(25): np[i] = (0,0,0)
    np.write()

def show(msg1, msg2=''):
    oled.clear(); oled.write_line(0, msg1)
    oled.write_line(1, msg2); oled.show()

print('박수로 춤을 지휘하세요!')
while True:
    show('CLAP DANCE', 'clap 1~3 times')
    n = count_claps(2000)
    if n == 0:
        continue
    n = min(n, 3)
    show('Dance #%d' % n)
    for _ in range(2): beat()       # 비트 점멸
    if   n == 1:
        motor.mecanum(8,60); sleep(0.4); motor.mecanum(2,60); sleep(0.4)
    elif n == 2:
        motor.drive('left', 60); sleep(1.2)
    else:
        motor.mecanum(6,60); sleep(0.4); motor.mecanum(4,60); sleep(0.4)
    motor.mecanum(5)
```

### ✅ 실행 & 확인
2초 안에 박수 1·2·3번 → 각각 다른 춤. OLED에 모드, LED는 비트에 맞춰 반짝.

> ### 🎯 도전과제 2-A — 소리 막대 미터
> 대기 중 OLED나 LED 매트릭스에 **현재 소리 크기를 막대로** 표시해 보세요.
> 힌트(1일차 sound_meter): `n = min(level // 80, 25)` 개의 LED를 켜기.

> ### 🎯 도전과제 2-B — 4박수 = 비상정지
> 박수 4번 이상이면 모든 동작을 멈추고 LED를 빨갛게, OLED에 `STOP!`을 띄우세요.

## 실습 3. 멀티모달 감정 로봇 🤖

> **여러 자극에 표정·몸짓으로 반응한다.** 과정 핵심 주제(멀티모달 인지)와 직결!

**감지** 버튼·박수·기울기(3가지 입력) → **판단** 감정 상태 → **동작** LED 표정 + 주행 반응 + OLED 대사

### STEP 1. 표정 패턴(LED) 만들기

5×5에 표정을 그립니다. 눈 2개(`6`, `8`)는 공통이고, 입 모양만 감정마다 바꿉니다.
아래 전체 코드의 `FACE` 딕셔너리를 그대로 사용하면 됩니다.

표정 3종 (실제 점등 칸):

| 감정 | 표정 | 점등 칸 |
|------|------|---------|
| happy 😊 | 웃는 눈·입 | 눈 `6,8` / 입 `15,19,21,22,23` |
| surprise 😮 | 큰 눈·동그란 입 | 눈 `6,8` / 입 `17,21,23,11,13` |
| dizzy 😵 | x자 눈 | 눈 `6,8` / 입 `16,18,22` |

### STEP 2. 세 가지 입력 동시에 읽기

```python
from machine import Pin, ADC
btn_a = Pin(36, Pin.IN)   # 쓰다듬기 → happy
btn_b = Pin(39, Pin.IN)
mic   = ADC(Pin(35)); mic.atten(ADC.ATTN_11DB)

def read_emotion(imu):
    # 1) 박수(큰 소리) → surprise
    s = [mic.read() for _ in range(40)]
    if max(s) - min(s) > 1800:
        return 'surprise'
    # 2) 버튼 A → happy
    if btn_a.value() == 0:
        return 'happy'
    # 3) 많이 기울임 → dizzy
    v = imu.get_values()
    if abs(v['AcX']) > 9000 or abs(v['AcY']) > 9000:
        return 'dizzy'
    return None    # 자극 없음
```

### STEP 3. 감정별 반응(주행 + 대사) 정의

```python
def react(motor, emo):

    elif emo == 'dizzy':

        motor.mecanum(5)
```

### STEP 4. 전체 합치기

```python
# emotion_bot.py  ----  실습 3 전체 코드
from ai_ponybot import i2c, PonyMotor, PonyOLED
from neopixel import NeoPixel
from machine import Pin, ADC
import mpu6050
from time import sleep

imu   = mpu6050.accel(i2c, 0x69)
motor = PonyMotor(i2c)
oled  = PonyOLED(i2c)
np    = NeoPixel(Pin(13, Pin.OUT), 25)
btn_a = Pin(36, Pin.IN)
mic   = ADC(Pin(35)); mic.atten(ADC.ATTN_11DB)

FACE = {
    'happy':    {'eyes': [6, 8], 'mouth': [15, 19, 21, 22, 23], 'c': (0, 12, 0)},
    'surprise': {'eyes': [6, 8], 'mouth': [11, 13, 17, 21, 23], 'c': (12, 8, 0)},
    'dizzy':    {'eyes': [6, 8], 'mouth': [16, 18, 22],          'c': (8, 0, 12)},
}
SAY = {'happy': 'I love it!', 'surprise': 'Whoa!', 'dizzy': 'So dizzy...'}

def show_face(emo):
    for i in range(25): np[i] = (0, 0, 0)
    f = FACE[emo]
    for i in f['eyes'] + f['mouth']:
        np[i] = f['c']
    np.write()

def read_emotion():
    s = [mic.read() for _ in range(40)]
    if max(s) - min(s) > 1800:
        return 'surprise'
    if btn_a.value() == 0:
        return 'happy'
    v = imu.get_values()
    if abs(v['AcX']) > 9000 or abs(v['AcY']) > 9000:
        return 'dizzy'
    return None

def react(emo):
    if emo == 'happy':
        motor.drive('left', 50); sleep(0.8); motor.drive('stop')
    elif emo == 'surprise':
        motor.mecanum(2, 60); sleep(0.5); motor.mecanum(5)
    elif emo == 'dizzy':
        motor.mecanum(4,50); sleep(0.3); motor.mecanum(6,50); sleep(0.3); motor.mecanum(5)

# 평상시 얼굴
for i in range(25): np[i] = (0,0,0)
np[6]=np[8]=(0,0,6); np[16]=np[17]=np[18]=(0,0,6); np.write()
oled.clear(); oled.write_line(0,'EMOTION BOT'); oled.write_line(1,'touch/clap/tilt'); oled.show()

print('자극을 줘 보세요! (버튼A=쓰다듬기, 박수, 기울이기)')
while True:
    emo = read_emotion()
    if emo:
        show_face(emo)
        oled.clear(); oled.write_line(0, emo.upper())
        oled.write_line(1, SAY[emo]); oled.show()
        react(emo)
        sleep(0.6)
    sleep(0.05)
```

### ✅ 실행 & 확인

## 실습 4. 자동 순찰 경비 로봇 🚨

> **스스로 사각형으로 순찰하다가 소리에 반응한다.** 5일차 프로젝트 맛보기!

**감지** 버튼(시작/정지)·마이크(침입) → **판단** 순찰/경계 모드 → **동작** 메카넘 순찰 + LED 경광등 + OLED

### STEP 1. 상태(모드) 설계

이 로봇은 **3가지 상태**를 오갑니다.

```
IDLE(대기) --A버튼--> PATROL(순찰) --큰 소리--> ALERT(경계) --B버튼--> IDLE
```

상태를 변수 하나로 관리하는 것이 핵심입니다.

```python
state = 'IDLE'   # 'IDLE' / 'PATROL' / 'ALERT'
```

### STEP 2. 경광등 LED (회전 효과)

```python
def beacon(np, color, step):
    # step을 늘려가며 한 줄(가로)씩 켜서 '도는' 느낌
    for i in range(25): np[i] = (0,0,0)
    row = step % 5
    for c in range(5):
        np[row*5 + c] = color
    np.write()
```

### STEP 3. 사각형 순찰 한 변 이동 + 소리 감시

순찰 중에도 소리를 계속 확인해야 하므로, **긴 sleep 대신 짧게 쪼개서** 감시합니다.

```python
def move_watch(motor, code, ms, check_sound):
    # code 방향으로 ms 동안 이동하되, 중간에 큰 소리가 나면 True 반환
    motor.mecanum(code, 55)
    t = 0
    while t < ms:
        if check_sound():
            motor.mecanum(5)
            return True       # 침입 감지!
        sleep(0.05); t += 50
    return False
```

### STEP 4. 전체 합치기

```python
# patrol_guard.py  ----  실습 4 전체 코드
from ai_ponybot import i2c, PonyMotor, PonyOLED
from neopixel import NeoPixel
from machine import Pin, ADC
from time import sleep

motor = PonyMotor(i2c)
oled  = PonyOLED(i2c)
np    = NeoPixel(Pin(13, Pin.OUT), 25)
btn_a = Pin(36, Pin.IN)    # 시작
btn_b = Pin(39, Pin.IN)    # 정지/해제
mic   = ADC(Pin(35)); mic.atten(ADC.ATTN_11DB)

INTRUDE = 2000   # 침입(큰 소리) 기준

def loud():
    s = [mic.read() for _ in range(40)]
    return (max(s) - min(s)) > INTRUDE

def beacon(color, step):
    for i in range(25): np[i] = (0,0,0)
    row = step % 5
    for c in range(5): np[row*5 + c] = color
    np.write()

def show(l0, l1):
    oled.clear(); oled.write_line(0, l0); oled.write_line(1, l1); oled.show()

def move_watch(code, ms):
    motor.mecanum(code, 55)
    t = 0
    while t < ms:
        if loud():
            motor.mecanum(5); return True
        sleep(0.05); t += 50
    return False

state = 'IDLE'
patrol = [8, 6, 2, 4]    # 전진→우횡→후진→좌횡 = 사각형
step = 0
show('GUARD READY', 'A=start')

print('A=순찰 시작, B=해제')
while True:
    if state == 'IDLE':
        beacon((0,8,0), step)            # 초록 천천히
        show('IDLE', 'A=start')
        if btn_a.value() == 0:
            state = 'PATROL'; sleep(0.3)

    elif state == 'PATROL':
        show('PATROL', 'watching...')
        beacon((0,10,0), step)
        code = patrol[step % 4]
        if move_watch(code, 800):        # 이동 중 소리 감지되면
            state = 'ALERT'
        if btn_b.value() == 0:
            motor.mecanum(5); state = 'IDLE'; sleep(0.3)

    elif state == 'ALERT':
        motor.mecanum(5)
        for k in range(6):               # 빨강 경광등 깜빡
            beacon((15,0,0), k)
            show('!! ALERT !!', 'INTRUDER')
            sleep(0.15)
        if btn_b.value() == 0:
            state = 'IDLE'; sleep(0.3)

    step += 1
    sleep(0.05)
```

### ✅ 실행 & 확인
A 누르면 사각형 순찰 + 초록 경광등. 손뼉 등 큰 소리 → 정지 + 빨강 경광등 + "ALERT". B로 해제.

> ### 🎯 도전과제 4-A — 경계 카운터
> 침입이 몇 번 감지됐는지 세서 OLED에 `Alerts: 3`처럼 누적 표시해 보세요.

> ### 🎯 도전과제 4-B — 서보 감시 카메라
> 순찰 중 `PonyServo`로 서보를 좌우(30↔150도)로 천천히 움직여
> '주변을 살피는 카메라' 연출을 추가하세요.

## 실습 5. 온도 탐사 로버 🌡️

> **구간을 이동하며 온도를 측정·기록한다.** 환경 탐사 로봇 컨셉.

**감지** 온도 센서 → **판단** 온도 구간 분류 → **동작** 정거장 이동 주행 + LED 색막대 + OLED 기록

### STEP 1. 온도 읽기

```python
import esp32
from time import sleep

while True:
    t = esp32.mcu_temperature()   # 칩 온도(℃) — 손으로 감싸면 오름
    print('%d C' % t)
    sleep(1)
```

> 칩 내부 온도라 주변보다 높게(보통 45~55℃) 나옵니다. **절대값보다 변화**에 주목하세요.
> 손으로 감싸면 올라가고, 식히면 내려갑니다. 기준선은 측정해 보고 정하세요.

### STEP 2. 온도 → 색 막대(LED) 변환

기준 온도 대비 낮음(파랑)·보통(초록)·높음(빨강)으로 5칸 막대를 칠합니다.

```python
def temp_bar(np, t, base):
    diff = t - base               # 기준선 대비 차이
    level = max(0, min(5, diff))  # 0~5칸
    if   diff <= 1: col = (0, 0, 12)   # 시원=파랑
    elif diff <= 3: col = (0, 12, 0)   # 적정=초록
    else:           col = (14, 0, 0)   # 더움=빨강
    for i in range(25): np[i] = (0,0,0)
    for r in range(level):             # 아래에서 위로 채우기
        for c in range(5):
            np[(4-r)*5 + c] = col
    np.write()
```

### STEP 3. '정거장' 이동 — 전진→정지→측정 반복

```python
def go_next_station(motor):
    motor.mecanum(8, 50); sleep(1.0)   # 1초 전진
    motor.mecanum(5)                   # 정지(측정 지점)
```

### STEP 4. 전체 합치기

```python
# temp_rover.py  ----  실습 5 전체 코드
from ai_ponybot import i2c, PonyMotor, PonyOLED
from neopixel import NeoPixel
from machine import Pin
import esp32
from time import sleep

motor = PonyMotor(i2c)
oled  = PonyOLED(i2c)
np    = NeoPixel(Pin(13, Pin.OUT), 25)
btn_a = Pin(36, Pin.IN)   # 시작
btn_b = Pin(39, Pin.IN)   # 정지

def read_temp():
    return esp32.mcu_temperature()

def label(diff):
    if   diff <= 1: return 'COOL'
    elif diff <= 3: return 'OK'
    else:           return 'HOT'

def temp_bar(t, base):
    diff = t - base
    level = max(0, min(5, diff))
    if   diff <= 1: col = (0,0,12)
    elif diff <= 3: col = (0,12,0)
    else:           col = (14,0,0)
    for i in range(25): np[i] = (0,0,0)
    for r in range(level):
        for c in range(5):
            np[(4-r)*5 + c] = col
    np.write()

# 기준선: 시작할 때 현재 온도를 base로 잡음
base = read_temp()
oled.clear(); oled.write_line(0,'TEMP ROVER')
oled.write_line(1,'base=%dC A=go' % base); oled.show()
print('A=탐사 시작 (base=%dC)' % base)

# A 버튼 대기
while btn_a.value() == 1:
    sleep(0.05)

station = 0
while True:
    station += 1
    motor.mecanum(8, 50); sleep(1.0); motor.mecanum(5)   # 다음 정거장으로 이동
    sleep(0.3)

    t = read_temp()
    temp_bar(t, base)
    oled.clear()
    oled.write_line(0, 'Station %d' % station)
    oled.write_line(1, '%dC  %s' % (t, label(t - base)))
    oled.show()
    print('Station %d: %dC (%s)' % (station, t, label(t - base)))
    sleep(1.5)

    if btn_b.value() == 0:        # B=탐사 종료
        motor.mecanum(5)
        oled.clear(); oled.write_line(0,'DONE'); oled.write_line(1,'%d stations' % station); oled.show()
        break
```

### ✅ 실행 & 확인
A로 시작 → 1초씩 전진하며 정거장마다 온도 측정. LED 막대 색·높이와 OLED에 정거장 번호·온도·상태. 측정 중 손을 칩에 대면 막대가 빨갛게 올라갑니다. B로 종료.

> ### 🎯 도전과제 5-A — 최고온 지점으로 복귀
> 가장 높은 온도가 나온 정거장 번호를 기억했다가, 종료 시 OLED에
> `Hottest: #3` 처럼 표시해 보세요. (변수에 max 갱신)

> ### 🎯 도전과제 5-B — 임계 초과 시 경보
> 온도가 base+4℃를 넘으면 즉시 정지하고 LED를 모두 빨강, OLED에 `OVERHEAT!`를 띄우세요.

## 부록. 자주 막히는 부분 (Troubleshooting)

| 증상 | 원인 | 해결 |
|------|------|------|
| LED가 안 켜짐 | `np.write()` 빠짐 | `np[i]=...` 뒤에 **반드시 `np.write()`** |
| `mpu6050` ImportError | 라이브러리 미업로드 | 보드에 `mpu6050.py` 업로드 |
| 가속도 값이 이상함 | 주소 틀림 | `imu = mpu6050.accel(i2c, 0x69)` 확인 |
| 모터가 안 움직임 | i2c 객체 분리 | `ai_ponybot`의 `i2c` 하나를 공유 |
| 기울임 방향이 반대 | 보드 장착 방향 | 비교 부등호(`>`↔`<`) 뒤집기 |
| 박수가 안 잡힘/너무 민감 | 임계값 | `THRESH`/`INTRUDE` 값 조절 |
| 멈추지 않음 | 무한 루프 | Thonny **Ctrl+C** 또는 Stop |
